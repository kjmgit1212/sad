Purpose of this Document

The Identity Manager provisioning platform is designed with extensibility as a primary goal. Many of the architectural components have been built using frameworks that can make use of custom extensions. This document describes how the shared access authorization engine can be extended by plugin custom authorization providers. The ability to plugin custom providers is provided through the use of a defined application programming interface (API). This document describes the Shared Access Authorization API. It also describes how to configure the custom providers.

Overview

Clients can use the Shared Access Authorization API to plugin one or more custom authorization providers. When there are multiple providers configured, the shared access authorization engine combines the results returned from all providers in the following manner:

  • Uses "Union" to combine the result of authorized shared accesses returned from multiple providers.
  • Uses "Or" to combine the boolean return of checkout authorization.

The Shared Access Authorization API consists of

  • An interface any custom provider must implement
  • A search filter class which defines the search attributes for the authorized shared accesses
  • A mediator class that invokes the authorization from all providers configured
  • An exception class which represents an error condition during authorization

The API classes can be found in the com.ibm.itim.pim.authority, classes in com.ibm.itim.dataservices.model.pim and com.ibm.itim.dataservices.model are also referenced as part of the API parameters.

API Description

The focal class in the API is the SharedAccessAuthorizationStrategy. This interface defines two authorization methods:

  • Find the authorized shared accesses for a given user with a search filter
  • Determine if a given user is authorized to check out a shared access
A custom authorization provider must implement this interface.

CredentialComponent is an interface representing a credential or credential pool. A authorization provider returns the authorized shared accesses as a collection of CredentialComponent. The provider can either construct CredentialComponent with Credential and CredentialPool, or provide its own implementation for CredentialComponent.

SharedAccessSearchFilter defines the search attributes for the authorized shared accesses. The search attributes include a list of predefined attributes, plus a generic attribute and value map for any additional search attributes which is provided for extensibility. A custom provider should filter the shared accesses with the search attribute values. SearchParameter defines the search size limit and scope. The scope is applicable only if SharedAccessSearchFilter specifies the match string for the business unit name; otherwise, the scope is ignored.

SharedAccessAuthority serves as a mediator in the shared access authorization engine. It calls SharedAccessAuthorizationProviderFactory to get all the providers that are configured in pim.properties and invokes the authorization method from all providers in sequence, until the maximum size limit reached or until one provider return boolean value "true". Below show an example of registering one custom authorization provider, and an example of registering one custom authorization provider together with the default authorization provider provided by Identity Manager: 

sa.authority.provider=examples.pim.SampleSAAuthrozationProvider


sa.authority.provider=examples.pim.SampleSAAuthrozationProvider,com.ibm.itim.pim.authority.store.SharedAccessPolicyEvaluator

If there are no providers configured in the properties file, SharedAccessAuthrozationProviderFactory return the default role-based shared access policy authorization provider, named SharedAccessPolicyEvaluator.

SharedAccessAuthorityException represents an error condition during authorization.
