Compiling and Running the Policy Analysis API Example

Compiling the Policy Analysis API Example

See Building the Examples for general instructions.

Preparing to Run the Example

Properties values associated with running the example are contained in the example.properties file. Before attempting to run the example, you must update the values in the example.properties file appropriately. See Updating the property files.

Running the Example

After compiling the example and setting the values in the example.properties file, run the example by running the runExample.cmd or runExample.sh file.

The example accepts the following input:

Argument

Description

getEntitlement

Obtains and prints the entitlement for the person and service specified in the example.properties file.

getEntitlements

Obtains the prints the entitlements for the person specified in the example.properties file.

getProvisioningParameters

Obtains the prints the provisioning parameters for the person and service specified in the example.properties file.

getProvisioningPolicies

Obtains the prints the provisioning policies for the role specified in the example.properties file. The provisioning policies returned do not include wildcard provisioning policies.

getProvisioningPoliciesWildcard

Obtains the prints the provisioning policies for the role specified in the example.properties file. The provisioning policies returned include wildcard provisioning policies.

None, one, or multiple arguments can be specified on a single invocation of the runExample.cmd or runExample.sh file. If no arguments are specified, all of the aforementioned functions are performed.

Running the Examples

To perform all of the functions associated with the example:

    runExample

To obtain and print the entitlement for the person and service specified in the example.properties file:

    runExample getEntitlement

To obtain the print the entitlements for the person specified in the example.properties file, and to obtain and print the provisioning policies for the role specified in the example.properties file (not including wildcard provisioning policies):

    runExample getEntitlements getProvisioningPolicies
