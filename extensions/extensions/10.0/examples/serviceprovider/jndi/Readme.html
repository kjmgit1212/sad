JNDI Service Provider Examples

The files in the JNDI directory implement a JNDI connector example. The JNDI connector example comes with configuration files that can be installed on the ISIM directory server to work without much effort. The JNDI connector, however, can be configured to use different schemas and can use different directory servers by updating these configuration files (see the Connector API documentation for more details). The only limitations of the connector is that it can’t support inheritance in the schema (other than from top), the user id attribute in the schema must be named erUid, and the password attribute must be named userPassword.

To run the example, first make sure the example code has been built and integrated with the ISIM platform (see Building the Examples). Then register the connector and its associated schema by following these steps.

To install the example, run the build file (build.bat for windows and build.sh for unix) found in the serviceprovider directory. The build file will create a jndisampleservice.jar file in the same directory. (You will need to set the ANT_HOME_LIB variable first.)

Now log in to ISIM as ITIM Manager and select Configure System > Manage Service Types. Press the Import button and select jndisampleservice.jar.

From this point, you can create an instance of the JNDI Sample Service registered by the connector and provision it. To properly configure an instance of the JNDI Sample Service, the following service attributes must be set correctly:

erJNDIFactory – Class name of a JNDI service provider. You can use the java.naming.factory.initial property in the enRoleLDAPConnection.properties file to use ISIM’s JNDI service provider.

erJNDIProviderURL – URL to the directory server to provision. You can use the java.naming.provider.url property in the enRoleLDAPConnection.properties file to use ISIM’s directory.

erJNDIPrincipal – User id to authenticate as to the directory server. You can use the java.naming.security.principal property in the enRoleLDAPConnection.properties file to authenticate as the ISIM system.

erJNDICredentials – Password to authenticate with to the directory server. You can use the java.naming.security.credentials property in the enRoleLDAPConnection.properties to authenticate as the ISIM system.

erJNDIRoot – Root node in the directory tree to search from and to add users to. This value must be the distinguished name of an existing object in the directory.
