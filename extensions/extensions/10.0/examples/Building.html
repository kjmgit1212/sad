Building The Examples

 


Setting up the Build Environment

Prior to compiling examples, complete the following steps.

  • Set WAS_HOME environment variable to the WebSphere Application Server home directory.
  • Add the wink.1.x.jar file to the extensions/lib directory. You can download the JAR file from https://wink.apache.org/

Compiling the Examples

Compiling the Examples

Open a command prompt and change directories to extensions/{RELEASE_VERSION}/examples. To compile the source code, type build on Windows, or build.sh on UNIX. This command compiles the Java classes into the following:

  • extensions/{RELEASE_VERSION}/lib/examples.jar
  • extensions/{RELEASE_VERSION}/examples/self_care/itim_expi.war
  • extensions/{RELEASE_VERSION}/examples/selfregistration/sr.war
  • extensions/{RELEASE_VERSION}/examples/ws/SSOSample/itim_ws.war

To view help for the build file, type build help on Windows, or ./build.sh help on UNIX.

Updating the Application Server's Classpath

Some examples require that examples.jar is available on the application server's classpath. The following link describes how to accomplish this: Adding examples.jar to Classpath
