Summary

The examples in this tutorial are aimed to give the user experience and understanding of the process required to use (IBM Directory Integrator) IDI to import identity information into IBM Security Identity Manager (ISIM) and to manage accounts in the ISIM data store on external resources using IDI. IDI is required to run these examples.

Overview

The examples presented here are intended to start users using IDI to import data into ISIM. Several examples of scenarios are presented where the new tools would be used in the place of previously used custom solutions or tools where use was difficult. These include:

  1. Using ISIM's reconciliation mechanism to query IDI for an identity feed. Previously, a custom program would have been required to format data from the original source into a Directory Services Markup Language (DSML) document.
  2. Using IDI to act as an end-point for managing accounts in an Lightweight Directory Access Protocol (LDAP) server. Previously, the LDAP-X end-point would have been necessary.
  3. Using IDI to feed changes in identity information into ISIM. Previously, a custom Java program using the Direcotry Access Markup Language (DAML) Java Naming and Directory Interface (JNDI) provider would have been required.

The examples can be worked through while reading the document IDI Directory Manager and IBM Security Identity Manager Integration, which documents the features available.
